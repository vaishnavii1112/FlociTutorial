# Build a Local Amazon S3 Workflow with Floci and Python

In this tutorial, we'll build a complete Amazon S3 workflow that runs entirely on our local machine using **Floci**, a free, open-source AWS emulator. By emulating the Amazon S3 API locally, Floci allows us to develop, test, and experiment with cloud applications without an AWS account or incurring any cloud costs.

Throughout this tutorial, we'll create and manage S3 buckets, enable object versioning, upload objects with metadata and tags, perform multipart uploads, generate pre-signed URLs, configure lifecycle policies, and process S3 event notifications using Amazon SQS. Finally, we'll build a simple S3-backed data lake by storing partitioned datasets and generating an aggregate report.

Because Floci implements the same API responses that `boto3`, the AWS CLI, and Terraform expect, everything you write here transfers directly to real AWS by changing a single endpoint setting.

## Starting Floci and Configuring the boto3 S3 Client

Before interacting with Amazon S3, we first need to start the Floci emulator. You can either install and start Floci using the CLI (recommended) or run it directly as a Docker container.

Install the required Python packages:

```bash
pip3 install boto3
```

### Step 1: Start Floci

**Option A (Recommended): Install and start Floci using the CLI**

```bash
curl -fsSL https://floci.io/install.sh | sh
floci start
```

**Option B: Run Floci using Docker**

```bash
docker run -d --name floci -p 4566:4566 floci/floci:latest
```

### Step 2: Configure the AWS SDK

Next, configure the AWS environment variables so that `boto3` connects to the local Floci instance instead of AWS.

```bash
export AWS_ENDPOINT_URL=http://localhost:4566
export AWS_DEFAULT_REGION=us-east-1
export AWS_ACCESS_KEY_ID=test
export AWS_SECRET_ACCESS_KEY=test
```

Note: Floci accepts any non-empty AWS credentials. The values test/test are a common convention used by local AWS emulators and are sufficient for local development.

### Step 3: Create the Python Application

Create a new Python file named `s3_tutorial.py` (or any name of your choice) and copy the following code into it. This script initializes a reusable `boto3` session, configures the S3 and SQS clients to communicate with the local Floci instance, and defines a few helper functions that will be used throughout the rest of the tutorial.

```python
import io
import os
import json
import time
import hashlib
import textwrap
from datetime import datetime, timezone

import boto3
from botocore.config import Config
from botocore.exceptions import ClientError, EndpointConnectionError

FLOCI_ENDPOINT = os.environ.get("AWS_ENDPOINT_URL", "http://localhost:4566")
REGION = os.environ.get("AWS_DEFAULT_REGION", "us-east-1")

# Floci accepts any non-empty credentials; "test"/"test" is the convention.
SESSION = boto3.session.Session(
    aws_access_key_id=os.environ.get("AWS_ACCESS_KEY_ID", "test"),
    aws_secret_access_key=os.environ.get("AWS_SECRET_ACCESS_KEY", "test"),
    region_name=REGION,
)

# Local emulators need path-style addressing (http://host:4566/bucket/key),
# not virtual-host style (http://bucket.host:4566/key).
S3_CONFIG = Config(
    signature_version="s3v4",
    s3={"addressing_style": "path"},
    retries={"max_attempts": 3, "mode": "standard"},
)

def client(service):
    return SESSION.client(service, endpoint_url=FLOCI_ENDPOINT, config=S3_CONFIG)

s3 = client("s3")
sqs = client("sqs")

def wait_for_floci(timeout=60):
    deadline = time.time() + timeout
    while time.time() < deadline:
        try:
            s3.list_buckets()
            print(f"Floci is ready at {FLOCI_ENDPOINT}")
            return
        except (EndpointConnectionError, ClientError):
            time.sleep(1)
    raise RuntimeError(f"Floci did not become ready at {FLOCI_ENDPOINT}")

def sha256_hex(data: bytes) -> str:
    return hashlib.sha256(data).hexdigest()

def banner(title: str):
    print("\n" + "=" * 68)
    print(title)
    print("=" * 68)

wait_for_floci()
```

Save the file and run it using:

```bash
python3 s3_tutorial.py
```

If Floci is running correctly, you should see output similar to:

```text
Floci is ready at http://localhost:4566
```

This confirms that your Python application can successfully communicate with the local Floci instance in the code above we explicitly select path-style S3 addressing, add SigV4 signing, and configure sensible retries. The `wait_for_floci` helper polls `list_buckets` until the emulator answers, which makes the script safe to run immediately after the container starts.
The remaining sections of this tutorial build upon this same script, so continue adding each subsequent code block to the end of s3_tutorial.py.

## Creating a Bucket, Enabling Versioning, and Uploading Objects

In this section, you'll create your first Amazon S3 bucket, enable object versioning, and upload two versions of the same object with custom metadata and tags.

### Step 4: Add the Bucket Creation and Object Upload Code

Append the following code to the end of `s3_tutorial.py`.

```python
BUCKET = "floci-demo-datalake"

banner("Create bucket + enable versioning")

def ensure_bucket(name, object_lock=False):
    try:
        kwargs = {"Bucket": name}
        # us-east-1 must NOT send a LocationConstraint; every other region must.
        if REGION != "us-east-1":
            kwargs["CreateBucketConfiguration"] = {"LocationConstraint": REGION}
        if object_lock:
            kwargs["ObjectLockEnabledForBucket"] = True
        s3.create_bucket(**kwargs)
        print(f"Created bucket: {name}")
    except ClientError as e:
        code = e.response["Error"]["Code"]
        if code in ("BucketAlreadyOwnedByYou", "BucketAlreadyExists"):
            print(f"Bucket already exists: {name}")
        else:
            raise

ensure_bucket(BUCKET)

s3.put_bucket_versioning(
    Bucket=BUCKET,
    VersioningConfiguration={"Status": "Enabled"},
)

status = s3.get_bucket_versioning(Bucket=BUCKET).get("Status")
print("Versioning status:", status)

def put_text_object(key, text, metadata=None, tags=None):
    body = text.encode("utf-8")
    kwargs = {
        "Bucket": BUCKET,
        "Key": key,
        "Body": body,
        "ContentType": "text/plain",
        "ServerSideEncryption": "AES256",
        "Metadata": metadata or {},
        "ContentMD5": None,  # let boto3 skip; Floci validates the SigV4 signature
    }
    kwargs = {k: v for k, v in kwargs.items() if v is not None}
    if tags:
        kwargs["Tagging"] = "&".join(f"{k}={v}" for k, v in tags.items())
    resp = s3.put_object(**kwargs)
    print(f"PUT {key:32s} version={resp.get('VersionId')} etag={resp['ETag']}")
    return resp

put_text_object(
    "notes/readme.txt",
    "Floci serves the real S3 API locally. This object is server-side encrypted.",
    metadata={"author": "tutorial", "stage": "draft"},
    tags={"team": "data", "pii": "false"},
)

# Upload a second version of the same key to exercise versioning.
put_text_object(
    "notes/readme.txt",
    "Second revision of the file, stored as a distinct S3 version.",
    metadata={"author": "tutorial", "stage": "reviewed"},
    tags={"team": "data", "pii": "false"},
)
```
### Step 5: Save and Run the Application

After appending the code, save the file and run:

```bash
python3 s3_tutorial.py
```

You should see output similar to:

```text
====================================================================
Create bucket + enable versioning
====================================================================
Created bucket: floci-demo-datalake
Versioning status: Enabled

PUT notes/readme.txt version=...
PUT notes/readme.txt version=...
```

In this section, you created a versioned Amazon S3 bucket and uploaded two versions of the same object. Versioning allows Amazon S3 to preserve previous versions of an object instead of overwriting them, making it easier to recover from accidental modifications or deletions.

The helper function `ensure_bucket()` creates the bucket if it does not already exist and safely handles the case where the bucket has already been created, allowing the script to be executed multiple times without errors. It also accounts for the special requirement that buckets created in the `us-east-1` region must not specify a `LocationConstraint`, while buckets in all other AWS regions must.

The `put_text_object()` helper uploads an object together with custom metadata, object tags, and server-side encryption (`AES256`) in a single API call. By uploading the same object twice, you've created two distinct versions of the object, which you'll explore in a later section of this tutorial.

With the bucket configured and versioning enabled, let's look at how Amazon S3 efficiently uploads large files using the Multipart Upload API.

## Streaming a Large Object with the S3 Multipart Upload API

Large files are usually uploaded to Amazon S3 using the Multipart Upload API. Instead of uploading one large file in a single request, the file is divided into smaller parts. Each part is uploaded separately, and S3 combines the parts into one final object.

### Step 6: Add the Multipart Upload Code

Append the following code to the end of `s3_tutorial.py`.

```python
banner("Multipart upload")

MULTIPART_KEY = "datasets/large-events.log"
PART_SIZE = 5 * 1024 * 1024  # 5 MiB, the S3 minimum part size (except the last part)
TOTAL_PARTS = 3

def synth_part(index: int) -> bytes:
    line = f"event_id,ts,payload -- part {index}\n".encode("utf-8")
    # Pad each part up to PART_SIZE with deterministic, compressible content.
    filler = (f"row-{index:03d}," * 64 + "\n").encode("utf-8")
    buf = bytearray(line)
    while len(buf) < PART_SIZE:
        buf.extend(filler)
    return bytes(buf[:PART_SIZE])

mpu = s3.create_multipart_upload(
    Bucket=BUCKET,
    Key=MULTIPART_KEY,
    ContentType="text/csv",
    ServerSideEncryption="AES256",
)
upload_id = mpu["UploadId"]
print("Started multipart upload:", upload_id)

parts = []
try:
    for i in range(1, TOTAL_PARTS + 1):
        chunk = synth_part(i)
        resp = s3.upload_part(
            Bucket=BUCKET,
            Key=MULTIPART_KEY,
            PartNumber=i,
            UploadId=upload_id,
            Body=chunk,
        )
        parts.append({"PartNumber": i, "ETag": resp["ETag"]})
        print(f"Uploaded part {i}: {len(chunk):,} bytes  etag={resp['ETag']}")

    result = s3.complete_multipart_upload(
        Bucket=BUCKET,
        Key=MULTIPART_KEY,
        UploadId=upload_id,
        MultipartUpload={"Parts": parts},
    )
    print("Completed multipart object:", result["Key"])
except Exception:
    s3.abort_multipart_upload(Bucket=BUCKET, Key=MULTIPART_KEY, UploadId=upload_id)
    raise

head = s3.head_object(Bucket=BUCKET, Key=MULTIPART_KEY)
print(f"Final object size: {head['ContentLength']:,} bytes")
print("Encryption:", head.get("ServerSideEncryption"))
```


```text
Uploaded part 2: 5,242,880 bytes  etag=...
Uploaded part 3: 5,242,880 bytes  etag=...
Completed multipart object: datasets/large-events.log
```

### What Happened?

In this section, you uploaded a large object to Amazon S3 using the Multipart Upload API.

The script first creates a synthetic log file in three separate parts. Each part is 5 MiB because Amazon S3 requires every multipart upload part, except the final part, to be at least 5 MiB.

The upload follows this sequence:

```text
Create multipart upload
        ↓
Upload part 1
        ↓
Upload part 2
        ↓
Upload part 3
        ↓
Complete multipart upload
        ↓
Final S3 object
```
Each uploaded part returns an `ETag`, which is stored in the `parts` list. Amazon S3 uses this list during `complete_multipart_upload()` to assemble the final object in the correct order.

The `try/except` block is important because if any part upload fails, the script calls `abort_multipart_upload()`. In real AWS, this prevents incomplete multipart uploads from remaining in the bucket and potentially increasing storage costs.

Finally, `head_object()` checks the completed object and confirms its final size and server-side encryption setting.

## Inspecting Versions, Listing Objects, and Reading Metadata

Now that the bucket contains multiple versions of an object, let's explore the information Amazon S3 stores for each object. In this section, you'll list object versions, retrieve a previous version of an object, inspect its metadata and tags, and list all objects stored in the bucket.

### Step 8: Add the Object Inspection Code

Append the following code to the end of `s3_tutorial.py`.

```python
banner("List versions and objects")

versions = s3.list_object_versions(Bucket=BUCKET, Prefix="notes/")
for v in versions.get("Versions", []):
    latest = "(latest)" if v["IsLatest"] else ""
    print(f"{v['Key']:24s} version={v['VersionId']:>12} size={v['Size']:>4} {latest}")

# Fetch a specific historical version and read its custom metadata.
oldest = min(versions["Versions"], key=lambda v: v["LastModified"])
obj = s3.get_object(Bucket=BUCKET, Key=oldest["Key"], VersionId=oldest["VersionId"])
print("\nOldest version body:")
print(" ", obj["Body"].read().decode("utf-8"))
print("Oldest version metadata:", obj["Metadata"])

# Read the tags attached to the current object.
tags = s3.get_object_tagging(Bucket=BUCKET, Key="notes/readme.txt")
print("Current tags:", {t["Key"]: t["Value"] for t in tags["TagSet"]})

banner("Paginated listing of the whole bucket")
paginator = s3.get_paginator("list_objects_v2")
total = 0
for page in paginator.paginate(Bucket=BUCKET):
    for item in page.get("Contents", []):
        total += 1
        print(f"{item['Key']:40s} {item['Size']:>10,} bytes")
print(f"Total objects: {total}")
```

## Generating Pre-Signed URLs and Attaching a Lifecycle Policy

Amazon S3 allows applications to temporarily share objects without exposing AWS credentials. In this section, you'll generate pre-signed URLs for uploading and downloading objects and configure a lifecycle policy to automatically manage objects stored in the bucket.

### Step 10: Add the Pre-Signed URL and Lifecycle Policy Code

Append the following code to the end of `s3_tutorial.py`.

```python
banner("Pre-signed URLs")

# A time-limited GET URL anyone can use without AWS credentials.
get_url = s3.generate_presigned_url(
    "get_object",
    Params={"Bucket": BUCKET, "Key": "notes/readme.txt"},
    ExpiresIn=3600,
)
print("Pre-signed GET URL:\n ", get_url)

# A pre-signed PUT URL for browser/direct uploads.
put_url = s3.generate_presigned_url(
    "put_object",
    Params={"Bucket": BUCKET, "Key": "uploads/from-presigned.txt"},
    ExpiresIn=3600,
)
print("Pre-signed PUT URL:\n ", put_url)

# Exercise the URLs with a plain HTTP client (no boto3, no credentials).
import urllib.request

payload = b"Uploaded through a pre-signed URL, zero SDK config required."
req = urllib.request.Request(put_url, data=payload, method="PUT")
with urllib.request.urlopen(req) as r:
    print("Pre-signed PUT status:", r.status)

with urllib.request.urlopen(get_url) as r:
    print("Pre-signed GET body:", r.read().decode("utf-8"))

banner("Lifecycle policy")
s3.put_bucket_lifecycle_configuration(
    Bucket=BUCKET,
    LifecycleConfiguration={
        "Rules": [
            {
                "ID": "expire-temp-uploads",
                "Filter": {"Prefix": "uploads/"},
                "Status": "Enabled",
                "Expiration": {"Days": 7},
            },
            {
                "ID": "clean-old-versions",
                "Filter": {"Prefix": ""},
                "Status": "Enabled",
                "NoncurrentVersionExpiration": {"NoncurrentDays": 30},
            },
        ]
    },
)
rules = s3.get_bucket_lifecycle_configuration(Bucket=BUCKET)["Rules"]
print("Active lifecycle rules:", [r["ID"] for r in rules])
```

## Wiring S3 Event Notifications to an SQS Queue

Amazon S3 can automatically notify other AWS services whenever objects are created or modified. In this section, you'll configure Amazon S3 to send object creation events to an Amazon SQS queue.

### Step 12: Add the Event Notification Code

Append the following code to the end of `s3_tutorial.py`.

```python
banner("S3 -> SQS event notifications")

QUEUE_NAME = "s3-object-created"
queue_url = sqs.create_queue(QueueName=QUEUE_NAME)["QueueUrl"]
queue_arn = sqs.get_queue_attributes(
    QueueUrl=queue_url, AttributeNames=["QueueArn"]
)["Attributes"]["QueueArn"]
print("Queue ARN:", queue_arn)

s3.put_bucket_notification_configuration(
    Bucket=BUCKET,
    NotificationConfiguration={
        "QueueConfigurations": [
            {
                "QueueArn": queue_arn,
                "Events": ["s3:ObjectCreated:*"],
                "Filter": {
                    "Key": {"FilterRules": [{"Name": "prefix", "Value": "incoming/"}]}
                },
            }
        ]
    },
)
print("Notification wired for prefix 'incoming/'")

# Trigger an event by uploading under the watched prefix.
put_text_object("incoming/order-1001.json",
                json.dumps({"order_id": 1001, "total": 42.5}))

# Poll SQS for the resulting S3 event.
messages = sqs.receive_message(
    QueueUrl=queue_url, MaxNumberOfMessages=5, WaitTimeSeconds=5
).get("Messages", [])

for m in messages:
    event = json.loads(m["Body"])
    for rec in event.get("Records", []):
        s3info = rec["s3"]
        print("Event:", rec["eventName"],
              "->", s3info["bucket"]["name"] + "/" + s3info["object"]["key"])
    sqs.delete_message(QueueUrl=queue_url, ReceiptHandle=m["ReceiptHandle"])

print(f"Consumed {len(messages)} S3 event message(s)")
```

### What Happened?

The script first creates an Amazon SQS queue and retrieves its Amazon Resource Name (ARN).

It then configures Amazon S3 to send `ObjectCreated` events whenever new objects are uploaded to the `incoming/` prefix.

Uploading a sample JSON file triggers an event notification, which is delivered to the SQS queue. The script then retrieves the event, displays its details, and removes the message from the queue.

This event-driven workflow is commonly used in production systems to trigger downstream processing whenever new data arrives in Amazon S3.

## Building an S3-Backed Data Lake

In this final example, you'll use Amazon S3 as a simple data lake. The script partitions a dataset into separate folders, reads the stored data back, calculates summary statistics, and stores the generated report in Amazon S3.

### Step 14: Add the Data Lake Code

Append the following code to the end of `s3_tutorial.py`.

```python
banner("S3-backed mini data lake")

# A tiny dataset we partition by category, one object per partition.
records = [
    {"category": "books", "sku": "B-1", "price": 12.0, "qty": 3},
    {"category": "books", "sku": "B-2", "price": 18.0, "qty": 1},
    {"category": "audio", "sku": "A-1", "price": 99.0, "qty": 2},
    {"category": "audio", "sku": "A-2", "price": 149.0, "qty": 1},
    {"category": "home", "sku": "H-1", "price": 24.5, "qty": 5},
]

def rows_for(category):
    return [r for r in records if r["category"] == category]

categories = sorted({r["category"] for r in records})
for cat in categories:
    partition = rows_for(cat)
    body = "\n".join(json.dumps(r) for r in partition).encode("utf-8")
    key = f"lake/orders/category={cat}/data.jsonl"
    s3.put_object(
        Bucket=BUCKET, Key=key, Body=body,
        ContentType="application/x-ndjson", ServerSideEncryption="AES256",
    )
    print(f"Wrote partition {key} ({len(partition)} rows)")

# Read every partition back and compute an aggregate report.
report = {}
resp = s3.list_objects_v2(Bucket=BUCKET, Prefix="lake/orders/")
for item in resp.get("Contents", []):
    data = s3.get_object(Bucket=BUCKET, Key=item["Key"])["Body"].read().decode()
    for line in filter(None, data.splitlines()):
        r = json.loads(line)
        stats = report.setdefault(r["category"], {"revenue": 0.0, "units": 0})
        stats["revenue"] += r["price"] * r["qty"]
        stats["units"] += r["qty"]

banner("Aggregate revenue report")
for cat in sorted(report):
    print(f"{cat:8s} revenue=${report[cat]['revenue']:>8.2f} "
          f"units={report[cat]['units']}")

# Persist the report itself back to S3 as the pipeline's output artifact.
s3.put_object(
    Bucket=BUCKET,
    Key=f"lake/reports/summary-{datetime.now(timezone.utc):%Y%m%dT%H%M%S}.json",
    Body=json.dumps(report, indent=2).encode("utf-8"),
    ContentType="application/json",
)
print("\nPipeline complete. Report written to lake/reports/.")
```

```bash
python3 s3_tutorial.py
```

You should see output similar to:

```text

====================================================================

Wrote partition lake/orders/category=books/data.jsonl
Wrote partition lake/orders/category=audio/data.jsonl
Wrote partition lake/orders/category=home/data.jsonl

====================================================================
Aggregate revenue report
====================================================================

audio revenue=...
books revenue=...
```

### What Happened?

The script partitions a small dataset by product category and stores each partition as a newline-delimited JSON file in Amazon S3.

Organizing data into folders such as `category=books` and `category=audio` follows a layout commonly used by analytics engines including Amazon Athena, Apache Spark, DuckDB, and Hive, allowing them to efficiently scan only the required partitions.

The script then reads every partition back, calculates the total revenue and units sold for each category, and writes the resulting report back to Amazon S3 as the final output of the pipeline.
